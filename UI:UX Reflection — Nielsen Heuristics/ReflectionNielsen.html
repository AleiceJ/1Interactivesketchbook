Norman + Nielsen

Heuristic number 5: Error Prevention feels most aligned with Norman's concerns in chapter 1. Nielsen states error prevention as a guideline for design that can be complicated. In the same way, Norman challanges us to design with simplicity in mind so that all ussers of every walk of life is able to use it. Both of these have Human Centered Design in mind. They both also think it is important to know what the error is. Without an error message, Norman says it causes confusion for the user. While Nielsen suggests a confirmation message before the user commits to a potential error. Preventing errors, or having error messages that help the user to understand the why or what of the error helps them to prevent future errors, and also teaches them how they could have prevented it.

Challenge Your Own Work

My micro-project clearly violates heuristic #5: error prevention. I feel the wording that I used to give the instructions can be easily confused which can cause errors. My instructions say, "Click the correct letter to complete the rainbow", and it shows letters on the screen. Some people may think you should click the letters on the screen, and not use your keyboard. Also, when the wrong letter is pressed, I don't have an error message, when the second wrong letter is pressed, there is still no error message, but the system resets. This may cause confusion. The user does not know what the did wrong or why it reset, they are guessing. Rules should be clearer to prevent errors. Another heuristic it violates is heuristic 1: Visibility of system status. When going to the next level, there is no level indicator, the user does not know what level they are on, or how many more levels they have to go to complete the game. This is important to have because it creates a system to be predicted which in turn builds trust.

Sketchbook Interface Audit

My Interactive Sketchbook could benefit from a makeover with these 10 heuristics in mind. There is no visible system status, just a page that opens up with links to my sketches. The navigation is consistent in that each "button" within the sketchbook takes you to that specific sketch. There are no reversible actions, you can use the browser's arrow to go back and forth between the "home" and previous page. Users aren't required to remember anything

The Hardest Heuristic

Honestly, I think the hardest heuristic to follow is number 5. I understand that you need to make your design as simple and easy to read so that anyone can use it, but there is more to it. I feel like to truly do that, you would need to run tests, have different users use it, and also predict potential problems and errors. It seems you would have to think beyond just design to prevent errors. I find it difficult because of conceptual tension.
