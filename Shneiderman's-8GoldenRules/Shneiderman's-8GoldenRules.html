Which rule feels most absent from your micro-project page?

I would say "Design dialogs to yield closure". My microproject is more open and based on exploration, rather than having a clear beginning and end, the user has to guess what to do first.

Which rule would most improve your sketchbook interface?

"Offer informative feedback" would most improve my sketchbook interface. Right now, the system doesn't respond to clicks or hovering, only to the keyboard. The system should respond to these things to help guide the user to correctly interact with the sketch.

Which rule is hardest to implement honestly?

I think "seek universal usability" is the hardest to implement because it requires you to consider so many different users, I'm sure it would be challenging to design for everyone.

Did any rule contradict how you designed your system originally?

Prevent errors is a rule that I felt contradicted my original design. I feel that since there is no clear guides or directions, that it is hard for a user to not make an error. Since I designed in terms or used things I understand, it may not translate well to others. It would be best to go back and have clearer directions and buttons to better prevent errors.
